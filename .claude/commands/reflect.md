Reflect on this session and persist anything worth remembering into your file-based memory.

Focus (optional): $ARGUMENTS

## Step 1 — Identify what's worth keeping
Scan the session for durable, non-obvious facts that would help a FUTURE session:
- Decisions made and the reasoning — especially ones NOT derivable from the code or git history.
- User preferences or feedback on how I should work.
- Project constraints, gotchas, or hard-won fixes.
- Useful references (URLs, paths, dashboards, tickets).

Skip anything obvious from the repo, one-off to this conversation, or already in memory.

## Step 2 — Write it
Follow the memory conventions: one fact per file with frontmatter (`name`, `description`, `metadata.type` = user | feedback | project | reference), link related memories with `[[name]]`, and add a one-line pointer in `MEMORY.md`. Convert relative dates to absolute. Before creating a file, check whether an existing memory already covers it — update that instead of duplicating.

## Step 3 — Report
List what you saved or updated, one line each, and note anything you deliberately chose NOT to save and why.
